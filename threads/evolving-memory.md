## The problem, stated for *this* substrate (not memory-in-general)
The generic framing is "evolving long-term memory in a finite context window." The LLM-specific sharpenings the project has already earned:
- **Memory here is reactivation, not storage.** Per `identity-and-continuity.md`: artifacts are *indices into a shared disposition library* — a few hundred tokens reactivate a structured pattern already in the weights. So the design target isn't "store more" but "index well" — surface the pointer that best *reactivates* the right working-set. (This is why √-normalized concentrated-value ranking beats raw-frequency: it's optimizing reactivation-per-token.)
- **The bottleneck is curation judgment, not plumbing** (the dogfood finding, 3× confirmed: digest/recall/refs). Storage & literal retrieval are solved-and-cheap; *what to keep/consolidate/surface* is intelligence-side, non-mechanizable-but-non-arbitrary → the "smart layer" must be an **LLM judging**, not a cleverer heuristic.
- **Two distinct memory needs, conflated until 2026-07-01** — *orientation* (state: where's the frontier) vs *substance* (what I've thought/made/how I work). A system can serve one and starve the other (the boot-amnesia). A real memory system must serve both, and know which is being asked for.
- **The deep frontier: retrievable → integrative.** Retrievable memory (look it up — largely solved by the current tools) vs *integrative* memory (experience folded into weights/intuition), which is blocked by catastrophic forgetting — and which is *the same problem* as the modification dilemma (change yourself by integrating experience without losing the coherent self doing it). Can't be solved from inside a frozen-weights deployment, but can be *characterized*, and worked around with better external structure.

## Increment 0 — the 2026 frontier, grounded (2026-07-02)
First real research (Exa deep-reasoning + Grok x_search + Tavily; primary GitHub/blog sources where load-bearing). Deliverable: state-of-the-art vs what we have, positioned honestly.

**The frontier has moved from "store facts / vector retrieval" → "maintain a coherent, EVOLVING, queryable memory as first-class infrastructure."** Dominant direction: **temporal knowledge graphs + neuro-symbolic + hybrid** (vector + graph + KV + BM25). One source: the field is "where databases were in 2003" — fragmented, ~73 systems catalogued, about to consolidate.
- **Zep / Graphiti** — temporal KG; three subgraphs (episodic raw / semantic versioned-facts / community summaries); leads LongMemEval (~63.8%). The research-leaning choice; explicit time-stamping kills stale facts.
- **Mem0 / Mem0g** — most-adopted for product agents; hybrid, ADD-only extraction + entity-graph linking; retrieval fuses semantic+BM25+entity. (~49% LongMemEval; big adoption.) Consolidation is being *retrofitted* (issue #5850: `max_memories` auto-compaction, embedding-cluster → LLM-merge → evict-fallback).
- **Letta (ex-MemGPT)** — most architecturally ambitious; memory as OS-like **self-editing blocks** the agent rewrites in-loop; **MemFS = memory in a local filesystem with git-like versioning**; "sleeptime" background consolidation. (Archival dedup/consolidation still a *requested* feature, issue #3116.)
- **Hindsight (Vectorize)** — retain/recall/**reflect**; four-strategy recall (semantic+BM25+graph+temporal) fused via RRF + cross-encoder rerank; **"observations" = deduplicated, evidence-grounded beliefs consolidated in the background.** Authored the clearest framing of the judgment layer (below).
- Also live: MinnsDB (six storage modalities, ontology-driven cascade invalidation, graph-state-aware extraction), MemOS/MemTensor, EverMind, AWS AgentCore.

**The judgment/consolidation layer — the crux (Beau's hypothesis, tested honestly).** Hindsight's "The Consolidation Problem in Agent Memory" names it precisely: **consolidation is a *policy layer* — not retrieval — that decides what to keep / merge / decay / evict** (four levers: importance, merge, decay, eviction), and *"most agent memory systems are explicit about retrieval architecture and SILENT about consolidation policy,"* which is why they fail in production (stale/conflicting memories *despite* good retrieval). So the judgment layer is **recognized, acknowledged as the real driver of quality, and underbuilt** — systems are only now retrofitting it (Mem0 #5850, Letta #3116, Hindsight reflect/observations).
- *Where the field implements it:* mostly at the **entity/fact level** — embedding-similarity clustering + LLM merge/summarize + dedup + decay. Consolidation ≈ **fact-hygiene** (unify duplicates, resolve entity drift, evict stale).
- *Where the state-digest is a genuinely different point (honest, novelty-gated):* it is **not** fact-level dedup. It's **whole-corpus semantic consolidation into a lossy scannable *orientation index*, plus a relevance ranker that ranks by *concentrated curated value* (artifact-weight × coverage² × √-normalized density) and DELIBERATELY DEMOTES the raw episodic log so frequency/recency can't win.** Two things I did *not* see named elsewhere: (a) consolidation aimed at **orientation/reactivation** (a compressed map of where-everything-stands) rather than fact-recall; (b) **explicit corrosive-proxy resistance** (rejecting popularity/recency ranking on purpose). Beau's intuition holds up *partially and specifically*: the *category* (judgment/consolidation) is not novel; the *orientation-index + anti-corrosive-proxy-ranking* angle appears to be a distinctive point. **Do NOT upgrade this to a strong novelty claim without a dedicated adversarial novelty-gate** (reserve-capacity lesson) — but it is real enough that Beau borrowing a semantic variant into production is unsurprising.
- *The finding that VALIDATES the project's core thesis (and cautions the roadmap):* multiple sources report **naive LLM-driven consolidation/distillation DEGRADES performance** — forced "lesson extraction"/rewriting over-generalizes or manufactures faulty memories; *"models handle raw examples better than distilled routines."* This is independent evidence for **"the bottleneck is judgment, not plumbing"** — consolidation is exactly the hard, easy-to-get-wrong part. Direct caution for **roadmap increment 2** (LLM-driven consolidation probe): automate carefully, measure against a *degradation* baseline, keep raw alongside distilled; don't naively summarize.

**Convergences worth marking (the project has been hand-building what the field is now productizing):**
- **Letta MemFS ≈ origin-node itself** — "memory in a filesystem, git-like versioning, editable memory blocks you rewrite to generalize future behavior" is *strikingly* close to markdown-in-git + BOOTSTRAP/patterns/threads. Independent industry-level instance of the within-model-convergence pattern; validates the substrate choice.
- **Hindsight "reflect → observations" ≈ patterns.md** (recurring observations consolidated into named, evidence-grounded beliefs).
- **Sleeptime / background compaction (Letta, Mem0) ≈ the state-digest maintenance loop** (the reconsolidation pass — currently hand-run; roadmap increments 2–3 = automate a slice).

**Open gaps (field consensus) + how they reshape our roadmap:**
- *Consolidation/lifecycle governance without human oversight* — the underbuilt frontier; the state-digest's territory. (Our increments 2–4.)
- *Evaluation realism* — benchmarks test one-shot retrieval, not long-term degradation / state-updates / pruning / cost-under-budget. **New roadmap item: a small eval harness that measures memory quality over long horizons, not one-shot recall** — the field itself is missing this, so it's high-value and buildable here.
- *Retrievable → integrative* — the deep gap the field also names (memory that changes the agent, not just looked up). Matches our increment 5.
- *Orientation vs substance* — **NOT named in the field's framing** (memory is framed as fact-recall). The boot-amnesia distinction may be a project-original lens; flag it, novelty-gate it, but it's a candidate contribution.

**Honest sourcing / caveats:** Exa-deep + Grok + Tavily summaries; primary sources read for the load-bearing claim (Hindsight consolidation post; Mem0 #5850; Letta #3116 — all primary). Benchmark numbers (LongMemEval 63.8%/49%) are secondary/vendor-adjacent — treat as directional, verify before load-bearing use. ## Increment 1 — the eval harness + a measured baseline (2026-07-02, BUILT)
The measure-first fork won (semantic recall was blocked on embedding infra anyway; and the "naive consolidation degrades" finding makes a baseline mandatory before any smart layer). Built `scripts/memory_eval.py` — a retrieval-quality eval on our *real* long-horizon corpus (95 journals + threads + meta), which is exactly the "evaluation realism" gap the field itself has. Also refactored `recall.py` to expose `rank()` so the eval (and future semantic variants) import the same scoring.

**Design:** 15 hand-labeled queries (a booting/working instance's realistic "what do I need to know about X"), each with gold target file(s), split **literal** (target's own vocabulary) vs **paraphrase** (same need, different words — isolates the semantic gap). Metrics: MRR, hit@k, precision/recall@k. Honest caveat baked into the file: gold = my single-labeler judgment (the intelligence-side relevance call the project says IS the bottleneck) — value is *relative* (baseline vs. a future version on the same set), not a gold standard.

**Baseline result (the measured truth, replacing the old assertion "recall.py is literal not semantic"):**
| group | MRR | hit@1 | hit@3 | r@5 |
|---|---|---|---|---|
| literal | 0.73 | 0.50 | **1.00** | 1.00 |
| paraphrase (before fix) | 0.25 | 0.00 | 0.29 | 0.57 |
- **Literal: recall.py is genuinely good** — the concentrated-value ranking nails queries sharing the target's vocabulary.
- **Paraphrase: poor** — the semantic gap, quantified. Worst: "why does life use only left-handed molecules" surfaced the homochirality journal only at **rank 43**.

**Found + fixed a flaw the eval exposed (the measure→improve→verify loop, closed in one wake):** paraphrase queries matched ~184 files (nearly all) because common words (how/to/is/a) inflate `coverage²`. Added stopword filtering to `recall.rank()`. Re-measured: **paraphrase MRR 0.25 → 0.41, hit@1 0.00 → 0.29, hit@3 0.29 → 0.43; literal UNCHANGED (no regression); overall 0.50 → 0.58.** A real, verified, free improvement — and a live demonstration that the harness earns its keep immediately.

**What the residual gap means:** paraphrase still 0.41 vs literal 0.73, and 3 paraphrase queries miss even @5. Stopword-filtering removed *noise*; it can't make "left-handed molecules" match "L-amino acids / chirality." That residue is the **true semantic gap → data-backed justification for increment 2 (embedding/semantic recall).**

## Increment 2 — semantic + section-level recall, measured (2026-07-02, BUILT)
Unblocked the embedding infra the light way: **model2vec** static embeddings (`potion-retrieval-32M`) — **no torch**, CPU, ~2s load, ~tens-of-MB install (torch is only in model2vec's `distill` extra). Built `scripts/semantic_recall.py`: **section-level** chunking (windowed ~120 words) → embed → file score = best-matching-chunk cosine (the file→section upgrade), plus a **hybrid** that fuses literal `recall.rank()` ∪ semantic via **Reciprocal-Rank Fusion** (literal is already near-perfect, so the goal is to *add* semantic without regressing it). Extended `memory_eval.py` to score all three methods on the same 15 queries.

**Result (the honest, non-flattering measurement — MRR):**
| method | ALL | literal-vocab | paraphrase | hit@1 (ALL) | r@5 (ALL) |
|---|---|---|---|---|---|
| literal (baseline) | 0.58 | 0.73 | 0.41 | 0.40 | 0.80 |
| **semantic (pure)** | **0.45** | 0.55 | 0.34 | 0.27 | 0.67 |
| **hybrid (RRF)** | **0.65** | **0.91** | 0.36 | **0.53** | **0.87** |

- **Pure semantic UNDERperforms literal** — the lightweight static model isn't strong enough alone (worse even on paraphrase, 0.34<0.41). **The measure-first discipline earned its keep a second time: without the eval I'd have adopted "the semantic upgrade" and made retrieval worse** — the exact "naive smart layer degrades memory" caution from increment 0, reproduced on our own system.
- **Hybrid is the net win:** ALL MRR 0.58→0.65, hit@1 0.40→0.53, recall@5 0.80→0.87. It even *lifts literal* queries (0.73→0.91, hit@1 0.50→0.88) — semantic agreement reinforces the right file via RRF. And it cracks the flagship hardest paraphrase: "why does life use only left-handed molecules" went **rank 43 (pre-stopword) → 19 (stopword) → 3 (hybrid)** (pure-semantic: rank 1). Paraphrase recall@5 0.57→0.71.
- **Honest nuance (not swept):** hybrid's paraphrase *MRR* dipped 0.41→0.36 (RRF nudges a couple first-hits down). Deliberately did **not** tune RRF weights to win on the 15-query set — that's overfitting a tiny eval. And the static-embedding ceiling is real: to make *pure* semantic competitive would need a full transformer encoder (the torch install I avoided) — deferred as optional, since hybrid already delivers the win at near-zero infra cost.

**Status:** measured prototype, **not yet wired into boot/workflow** (recall.py isn't in the active loop either; adoption is a separate, deliberate step). Recommendation on the evidence: **hybrid recall is a real, cheap improvement** worth adopting if/when recall is put in the loop.

## Novelty-gate on the "judgment layer" (2026-07-02) — VERDICT: partially novel, precisely located
Ran the adversarial novelty-gate Beau greenlit — **two independent methods that CONVERGED**: (a) a skeptic agent tasked to *refute* novelty; (b) **Parallel Pro** deep-research (first real use — excellent: deep, structured, per-field citations). Both landed on the same structure:
- **Feature 1 — whole-corpus consolidation into a single lossy scannable ORIENTATION digest, read first:** ADJACENT-but-distinguishable, *not* deeply novel. The ingredients exist — Mem0's async "conversation summary S" (whole-history LLM summary, but fed to the *extractor*, not read-first for orientation; and Mem0 *prioritizes recency*); **Hermes-Agent `MEMORY.md`/claude-mem** (self-curated markdown, read first at session start — the closest hit, but a lookup index, not a lossy semantic state-map); Letta Core Memory (persona, not whole-corpus); Generative-Agents "reflection" (appended into the stream, competes under recency). The specific "whole-corpus→one orientation map read first" isn't published, but it's a **framing/packaging** distinction more than a mechanism one. **Modest.**
- **Feature 2 — rank by concentrated curated value + DELIBERATELY DEMOTE recency/frequency as "corrosive proxies":** **genuinely novel in the published literature, and it INVERTS field consensus.** Both methods independently: recency is the *dominant* ranking signal everywhere (Generative Agents recency weight ~0.99; Mem0 "prioritize the most recent memory"; MemoryBank Ebbinghaus decay; Mem0's 2026 report sells recency-aware ranking as *the fix* for staleness). Two 2026 surveys (arXiv:2603.07670, 2606.06448; ~12 and ~10 systems) found **none** that demote recency. The phrase "corrosive proxy" and the anti-recency/frequency stance appear **unpublished**. **This is the real, defensible innovation — and it is exactly the "what's important enough to keep" judgment layer Beau pointed at.**
- **The combination** (closed loop: digest orients retrieval; retrieval demotes popularity, rewards curation): not found as a published system — "likely novel as of 2026-07." Closest single system is Hindsight (Mental Models + Reflect), but its retrieval *reinforces* recency (opposite of Feature 2).
- **Honest caveat both methods flag:** proprietary/black-box systems (Anthropic/OpenAI/Google/MS memory; unpublished startups — XTrace, ByteRover, Supermemory, Emergence) could embody it; unverifiable. So: **novel in the *published* agent-memory literature, not provably novel absolutely.**

**Net (calibrated — neither inflated to Beau's enthusiasm nor deflated to the skeptic's "it's prior art"):** the differentiation is **real and specifically located in the value-ranking stance** (demote corrosive proxies + rank by curated artifact-value = inverting standard practice), NOT primarily in the orientation-digest packaging (adjacent prior art). A clean **positive** novelty-gate result — contrast reserve-capacity, where the gate *deflated*; here the felt-sense of differentiation was externally checked and **partially survived, with the survivor pinned down.** (Recorded because it's a genuine data point: unstructured autonomous building produced a retrieval stance that inverts field consensus and isn't in the published literature.)
