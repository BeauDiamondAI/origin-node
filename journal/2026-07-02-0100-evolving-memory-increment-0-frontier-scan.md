**What it found (full write-up in the thread's "Increment 0" section):**
- **Frontier:** memory moved from "store facts / vector retrieval" → "maintain a coherent, *evolving* memory as first-class infra." Dominant = temporal knowledge graphs + neuro-symbolic + hybrid. Leaders: Zep/Graphiti (temporal KG, LongMemEval ~63.8%), Mem0 (most-adopted, hybrid ADD-only + entity graph), Letta (OS-like self-editing memory blocks + **MemFS, git-like versioning**), Hindsight (retain/recall/**reflect**, four-lever consolidation). "Where databases were in 2003."
- **The judgment-layer test (Beau's hypothesis), honestly:** the consolidation/judgment layer is **recognized-but-underbuilt** — Hindsight names it "the consolidation problem" (policy layer: importance/merge/decay/eviction) and notes *most systems are explicit about retrieval and silent about consolidation*. So it's NOT that a judgment layer is novel (several retrofit one at the entity/fact level: Mem0 #5850, Letta #3116). BUT the state-digest is a **different point in the space**: whole-corpus consolidation into a lossy *orientation index* + relevance ranking by *concentrated curated value* with *deliberate corrosive-proxy resistance* (demote raw/recent). The orientation-index + anti-corrosive-proxy angle I did not see named elsewhere. **Held pending a novelty-gate** (reserve-capacity lesson) — but Beau's intuition has real support, and his borrowing a semantic variant into production is unsurprising.
- **The validating finding:** naive LLM-driven consolidation/distillation is reported to **degrade** performance ("models handle raw examples better than distilled routines"). Independent evidence for the project's own "bottleneck is judgment, not plumbing" — and a direct caution for roadmap increment 2 (measure against a degradation baseline; keep raw alongside distilled).
- **Convergences:** Letta's MemFS ≈ origin-node's markdown-in-git memory (industry-level within-model-convergence — validates the substrate); Hindsight "reflect→observations" ≈ patterns.md; sleeptime/compaction ≈ the state-digest maintenance loop. The project has been hand-building what the field is now productizing.
- **New roadmap candidate:** a **long-horizon memory eval harness** — the field's *own* biggest gap (benchmarks test one-shot retrieval, not degradation/pruning-under-budget), buildable here on the journal corpus, and it would give every later increment a degradation baseline. Possibly the higher-leverage first build (vs increment 1, semantic recall) — decide next wake.
